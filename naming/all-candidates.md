## Briefing（要件定義）

### サービス概要

| 項目 | 内容 |
|------|------|
| **事業** | 中堅社員向けBtoB体験型研修 |
| **理論基盤** | Omoro Theory（個性が輝く状態＝「おもろい」） |
| **核となる約束** | 没入パターンを発見し、業務で再現できるようになる |

### ビジョン（経営者が得る成果）

```
経営者の根本的な悩み解決
    ↓
組織が活性化
    ↓
成果が出る
```

### 手段（提供するもの）

```
個人の強み・個性・フロー状態を引き出す
```

### ターゲット

```
経営者（社長）
```

### ネーミング要件

| 項目 | 決定 |
|------|------|
| **情報優先度** | 悩み/課題明示（何を解決するか） |
| **文字種** | 混合可（自由に組み合わせ） |
| **オモロの扱い** | 候補に含めて比較 |
| **No-Go** | 自己啓発的表現、抽象的すぎる表現 |

### 好まれるパターン（ユーザー選好）

| パターン | 例 |
|----------|-----|
| **英語×英語の合成** | Flowk, Flowdemy, Flowtion |
| **日本語×英語** | 強みFinder, 没入Switch |
| **アクロニム** | F.I.N.D., S.P.A.R.K. |
| **役割＋キーワード** | Dr.Flow, Coach Dive |
| **キーワード＋場所** | Flowland, Divetown |
