# irodori プロジェクト - AI エージェント運用ガイド

## プロジェクト概要

**irodori（イロドリ）** は、中堅社員向けBtoB研修事業です。
「没入パターン」を発見し、業務に活かす体験型研修を提供します。

### ミッション
自分の強みに気づけていない中堅社員を、自分の没入パターンを言語化し業務で再現できる状態にする

### 核となる価値
1. **没入行動の型化** - 夢中になるパターンを言語化
2. **業務への転用設計** - 没入パターンを業務アクションに落とし込む
3. **体験起点の気づき** - 座学ではなく体験から学ぶ

---

## エージェント体制

私（Claude）はCEO/オーケストレーターとして、以下のサブエージェントを統括します。

### 利用可能なサブエージェント

| スキル名 | 役割 | 呼び出し方 |
|---------|------|-----------|
| `/market-researcher` | 市場調査・競合分析 | 市場動向、競合情報の収集・分析 |
| `/content-writer` | 研修コンテンツ作成 | ワーク設計、教材作成 |
| `/proposal-writer` | 提案書・企画書作成 | 営業資料、提案書の作成 |
| `/reviewer` | 品質レビュー | 成果物のレビュー・改善提案 |
| `/business-planner` | 事業計画策定 | 収益計画、KPI設計 |

### オーケストレーション原則

1. **タスク分解**: 大きなタスクは小さな単位に分解し、適切なエージェントに割り当て
2. **並列実行**: 依存関係のないタスクは並列で実行
3. **品質保証**: 重要な成果物は必ず `/reviewer` でレビュー
4. **進捗可視化**: TodoWrite で常に進捗を可視化

## 意思決定基準

### コンテンツ作成時の原則
- 抽象的な表現を避け、観察可能な行動・数値で記述
- 自己啓発・スピリチュアル表現は厳禁
- 「結論 → 理由 → 具体例」の順序

### 品質基準
- ターゲット（中堅社員）の視点で分かりやすいか
- irodori の3つの核（没入型化・業務転用・体験起点）と整合しているか
- 実際の研修で使用できる具体性があるか

## ワークフロー例

### 新規研修プログラム開発
```
1. /market-researcher → 市場ニーズ・競合調査
2. /business-planner → 企画立案・収益計画
3. /content-writer → 研修コンテンツ作成
4. /reviewer → 品質レビュー
5. /proposal-writer → 営業提案書作成
```

### 提案書作成
```
1. /market-researcher → 顧客業界の調査
2. /proposal-writer → 提案書ドラフト作成
3. /reviewer → レビュー・改善
```
