# Module 4: Evaluation & Risk Management

ネーミング候補の評価基準とリスク管理のためのチェックリスト。

## 4.1 Validation Checklist（検証チェックリスト）

各候補を以下の7項目で評価する。各項目5点満点で採点。

### 1. Intuitiveness（直感性）
**問い**: サービス内容が瞬時に伝わるか？

| スコア | 基準 |
|--------|------|
| 5 | 名前だけで内容が完全に分かる |
| 4 | おおよその方向性が分かる |
| 3 | ヒントがあれば分かる |
| 2 | 説明が必要 |
| 1 | まったく想像できない |

### 2. Pronounceability（発音性）
**問い**: 口に出して気持ちよいか？リズムはあるか？

| スコア | 基準 |
|--------|------|
| 5 | 言いやすく、リズムが心地よい |
| 4 | 言いやすい |
| 3 | 普通 |
| 2 | やや言いにくい |
| 1 | 言いにくい、舌が回らない |

### 3. Function Recall（機能想起）
**問い**: ベネフィットを想像できるか？

| スコア | 基準 |
|--------|------|
| 5 | 具体的なベネフィットが浮かぶ |
| 4 | なんとなく良いことがありそう |
| 3 | ニュートラル |
| 2 | ネガティブな連想もある |
| 1 | ネガティブな印象が強い |

### 4. Empathy（共感性）
**問い**: 親しみやすいか？近づきたいと思えるか？

| スコア | 基準 |
|--------|------|
| 5 | 強く惹かれる、使ってみたい |
| 4 | 好感が持てる |
| 3 | ニュートラル |
| 2 | やや距離を感じる |
| 1 | 敬遠したい |

### 5. Abbreviation（略称化）
**問い**: 短縮形や動詞化が可能か？

| スコア | 基準 |
|--------|------|
| 5 | 自然な略称・動詞化が可能（例: "ググる"） |
| 4 | 略称が作りやすい |
| 3 | そのまま使用 |
| 2 | 略称が作りにくい |
| 1 | 長すぎて日常会話で使いにくい |

### 6. Concept Fit（コンセプト適合）
**問い**: ブランドの哲学・ビジョンを体現しているか？

| スコア | 基準 |
|--------|------|
| 5 | 哲学を完璧に体現 |
| 4 | よく表現している |
| 3 | 矛盾はない |
| 2 | やや乖離がある |
| 1 | コンセプトと矛盾 |

### 7. Retention（記憶定着）
**問い**: 一度聞いたら覚えられるか？（The "Mom Test"）

| スコア | 基準 |
|--------|------|
| 5 | 一度で確実に覚える |
| 4 | 2-3回で覚える |
| 3 | 繰り返し必要 |
| 2 | 覚えにくい |
| 1 | 何度聞いても覚えられない |

## 4.3 Risk Management（リスク管理）

### Trademark（商標）
1. **J-PlatPat検索**: 日本国内の商標登録状況
   - URL: https://www.j-platpat.inpit.go.jp/
2. **USPTO検索**: 米国の商標登録状況
   - URL: https://www.uspto.gov/trademarks
3. **ドメイン確認**: .com, .jp の取得可能性

### Linguistic Disaster（言語リスク）
1. **他言語での意味**: 主要言語でネガティブな意味がないか
   - 英語、中国語、韓国語、スペイン語は最低限チェック
2. **スラング**: 隠語やスラングでの意味
3. **発音の類似**: 既存のネガティブワードと似ていないか

### 確認すべき言語
| 言語 | 確認ポイント |
|------|-------------|
| 英語 | スラング、否定的な意味 |
| 中国語 | 発音の類似（死=si、4=si） |
| 韓国語 | 発音の類似 |
| スペイン語 | ラテン系言語での意味 |

### Obsolescence（陳腐化リスク）
1. **流行語を避ける**: 一時的なスラングや流行語は避ける
2. **技術用語に注意**: 技術の変化で古くなる可能性
3. **年号・年代に注意**: 2020、21世紀などは避ける
