# Module 3: Japanese Contextual UX

文字種が持つ「視覚的テクスチャ」と「意味」の制御。日本語ネーミングにおける独自の考慮事項を整理する。

## 3.1 Script Strategy（文字種戦略）

日本語には4つの文字種があり、それぞれが異なる認知的効果を持つ。

### ひらがな
| 属性 | 説明 |
|------|------|
| イメージ | Softness, familiarity, maternity, tradition |
| 適用業種 | 介護・ケアサービス、食品、子供向け、伝統工芸 |
| 心理効果 | 親しみやすさ、安心感、やさしさ |

**例**:
- ほっともっと（弁当）
- すかいらーく（ファミレス）
- あんしん（保険）

### カタカナ
| 属性 | 説明 |
|------|------|
| イメージ | Advanced, functional, speed, global |
| 適用業種 | IT、テクノロジー、美容、アパレル、スポーツ |
| 心理効果 | 先進性、スタイリッシュ、クール |

**例**:
- メルカリ（フリマ）
- クックパッド（レシピ）
- サイボウズ（グループウェア）

### 漢字
| 属性 | 説明 |
|------|------|
| イメージ | Trust, history, heavy, formal |
| 適用業種 | 金融、B2B、法律、医療、伝統産業 |
| 心理効果 | 信頼感、格式、重厚感 |

**例**:
- 三井住友（銀行）
- 野村證券（証券）
- 湖池屋（スナック）

### 英字・ローマ字
| 属性 | 説明 |
|------|------|
| イメージ | Global, sophisticated, modern |
| 適用業種 | グローバル企業、ラグジュアリー、IT |
| 心理効果 | 国際性、洗練、先進性 |

**例**:
- LINE
- ZOZO
- UNIQLO

### Warning（注意事項）
- **読めない英語は避ける**: ターゲットが発音できない英語は「ノイズ」になる
- **難読漢字は避ける**: 読み方が分からない漢字は認知負荷が高い
- **文字種の混在**: 意図的でない限り、混在は避ける（例外: PayPay）

## 3.2 Information Hierarchy（情報階層）

ネーミングに埋め込むべき情報の優先順位。

### 優先度順
1. **Target（Who）**: 誰のためのサービスか
2. **Problem（What to solve）**: 何を解決するか
3. **Benefit（Gain）**: 得られる価値は何か
4. **Format（Class, Agent, Salon）**: サービスの形態

### 組み込みパターン

#### パターンA: Target明示型
```
[ターゲット] + [ベネフィット/機能]
例: ママリ（ママ + 〜り）
```

#### パターンB: Problem明示型
```
[課題] + [解決]
例: 価格.com（価格の比較を解決）
```

#### パターンC: Benefit明示型
```
[得られる価値] + [修飾]
例: 楽天（楽しい + 天国）
```

#### パターンD: Format明示型
```
[サービス形態] + [領域]
例: クラスパス（Class + Pass）
```

## 3.3 日本語特有の技法

### 語呂合わせ・掛詞
- 日本語の同音異義語を活用
- 例: 居酒屋「酔虎伝」（すいこでん＝水滸伝）

### やまとことば活用
- 和語の柔らかさを活用
- 例: 「つながる」「かなえる」「ひらく」

### オノマトペ活用
- 擬音語・擬態語の活用
- 例: ぐるなび（ぐるぐる + ナビ）

### リダプリケーション（反復）
- 音の繰り返しによる記憶定着
- 例: メルカリカリ、PayPay

## 3.4 視覚的考慮事項

### 文字数バランス
- ロゴ化を想定した文字数（3-6文字が最適）
- 長すぎると視認性低下

### 字面のバランス
- 漢字の画数バランス
- カタカナの縦横バランス

### デジタル表示
- スマホでの表示を想定
- 小さいフォントでも識別可能か
