# Module 2: Ideation Patterns & Frameworks

サービス名を発想するための4つの基本フレームワークと、それぞれの特性・活用場面を解説する。

## 2.1 Descriptive（記述型）

サービスの内容や機能を直接的に表現するアプローチ。

### 特徴
- **メリット**: 即座に理解可能、説明コスト低
- **デメリット**: 差別化困難、商標取得困難
- **適用場面**: 新カテゴリ創出時、B2B、機能訴求重視

### パターン例
| パターン | 説明 | 例 |
|---------|------|-----|
| 機能直結型 | 何をするかを直接表現 | PayPal, YouTube |
| カテゴリ＋修飾語 | 業種＋特徴 | QuickBooks |
| ベネフィット型 | 得られる価値を表現 | Indeed |

### 発想プロンプト
1. このサービスは「何を」するか？
2. ユーザーは「何を」得るか？
3. 一言で説明すると？

## 2.2 Metaphor（比喩型）

抽象的な概念やイメージを借用して、サービスの本質を間接的に表現する。

### 特徴
- **メリット**: 感情喚起、記憶に残る、差別化可能
- **デメリット**: 理解に時間がかかる場合あり
- **適用場面**: ブランド構築重視、感情訴求、B2C

### パターン例
| パターン | 説明 | 例 |
|---------|------|-----|
| 自然物 | 自然界からの借用 | Apple, Amazon, Twitter |
| 神話・伝説 | 神話や伝説からの借用 | Nike, Oracle, Pandora |
| 動物 | 動物の特性を借用 | Jaguar, Puma, Firefox |
| 場所 | 地名・場所の借用 | Patagonia, Adobe |

### 発想プロンプト
1. このサービスを動物に例えると？
2. 神話の登場人物なら誰？
3. 自然現象で表すと？
4. どんな場所のイメージ？

## 2.3 Coined（造語型）

既存の言葉を組み合わせたり、完全に新しい言葉を創造する。

### 特徴
- **メリット**: 商標取得容易、唯一無二、グローバル展開容易
- **デメリット**: 意味伝達に時間がかかる、認知コスト高
- **適用場面**: グローバル展開、長期ブランド構築

### パターン例
| パターン | 説明 | 例 |
|---------|------|-----|
| 完全新規 | 意味のない音の組み合わせ | Kodak, Xerox |
| ラテン/ギリシャ語根 | 古典言語の語根活用 | Agilent, Verizon |
| 音象徴ベース | 音の印象から構築 | Zoom（素早さのZ音） |

### 発想プロンプト
1. 伝えたい属性を音で表すと？
2. ラテン語・ギリシャ語で関連する語根は？
3. 2-3音節で発音しやすい組み合わせは？

## 2.4 Blended（混成型）

2つ以上の単語や概念を融合させて新しい名前を作る。

### 特徴
- **メリット**: 意味が推測可能かつユニーク
- **デメリット**: 融合が不自然だと違和感
- **適用場面**: 複数の価値を表現したい場合

### パターン例
| パターン | 説明 | 例 |
|---------|------|-----|
| ポートマントー | 2語の融合 | Pinterest (Pin + Interest) |
| 頭字語 | 頭文字の組み合わせ | IBM, BMW |
| 接頭辞・接尾辞 | 既存語＋接辞 | Spotify (Spot + -ify) |
| かばん語 | 語の一部同士を結合 | Groupon (Group + Coupon) |

### 発想プロンプト
1. サービスを表す2つのキーワードは？
2. それらを融合させると？
3. 音の響きは自然か？

## 2.5 発想セッションの進め方

### Step 1: 発散フェーズ
- 各フレームワークで最低10個ずつ発想
- 批判禁止、量を重視
- 連想ゲーム的に広げる

### Step 2: 収束フェーズ
- 音象徴（Module 1）の観点で絞り込み
- 日本語UX（Module 3）の観点で調整
- 評価チェックリスト（Module 4）で検証

### Step 3: 磨き込み
- 候補を声に出して読む
- 第三者に聞かせて反応を見る
- 文字種（ひらがな/カタカナ/漢字/英字）を変えて試す
